# RULES

1. **No guessing** — Every claim must be backed by decompiled code, disassembly, strings, or cross-references from IDA. If something is unknown, say "unknown" and mark it for further investigation.

2. **No mistakes** — Verify every function name, address, class name, and call chain before writing it down. Cross-check with xrefs, callees, and caller lists.

3. **No rushing** — Slow and steady. Analyze one component at a time, fully, before moving to the next. It takes as long as it takes.

4. **RULES.md checkpoint rule** — Every time you compact context, re-read, or resume work, you MUST read RULES.md and CLAUDE.md before doing anything else. No exceptions.
